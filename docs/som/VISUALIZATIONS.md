# SOM Visualizations Guide

What each generated visualization shows and how to read it. All examples come
from an S-Curve benchmark run (15×15 hex map, 2D manifold bent in 3D — see
`RUN.md` for the benchmark battery). Replaces the former `EXAMPLES.md`.

Two families:

- **SOM maps & training plots** — rendered into `<results_dir>/visualizations/`
  during a run (`save_visualizations` / `save_training_plots`), or later via
  `som.visualization.render_results_dir()`.
- **Topology projections** — rendered on demand by
  `app/tools/plot_som_topology.py` into the results dir root; they verify that
  the map preserved the *structure of the data space*.

---

## Part 1 — SOM maps

### U-Matrix

![U-Matrix](examples/maps/u_matrix.png)

Mean distance of each neuron's weight vector to its grid neighbors (6 for hex,
4 for square — same neighbor convention as training, see `issues.md` #25).
**Dark valleys** = coherent regions of similar neurons (clusters); **light
ridges** = boundaries where adjacent neurons differ sharply (cluster borders).
The primary map for spotting cluster structure at a glance.

### Hit Map

![Hit Map](examples/maps/hit_map.png)

Number of samples assigned to each neuron (BMU counts, printed in red).
Read it for **map utilization**: a healthy organization spreads hits broadly;
large empty areas next to overloaded neurons mean the map size or training
schedule doesn't match the data. On the uniform-cube negative control this
map must look uniform.

### Distance Map (Neuron QE)

![Distance Map](examples/maps/distance_map.png)

Mean quantization error of the samples assigned to each neuron. **Bright
neurons represent their samples poorly** — either genuinely anomalous regions
or under-fitted parts of the map. Cross-reference with the Hit Map: high QE +
many hits = the map lacks resolution there; high QE + one hit = outlier
candidate.

### Dead Neurons Map

![Dead Neurons Map](examples/maps/dead_neurons_map.png)

Binary view: black = neuron never wins (no assigned samples). Scattered single
dead neurons are normal; **connected dead regions** signal a torn topology or
an oversized map (`dead_ratio` is one of the EA objectives).

### Cluster Map

![Cluster Map](examples/maps/cluster_map.png)

Each active neuron in its own color. Reminder of the clustering convention:
**a cluster = the set of samples assigned to one neuron** — no secondary
clustering algorithm runs on top (`RESULTS.md`).

### Component Planes

![Component x](examples/maps/component_x.png)
![Component y](examples/maps/component_y.png)
![Component z](examples/maps/component_z.png)

One plane per training column: the weight value of that feature across the
grid (numeric features are denormalized back to their real range). Read them
for **feature gradients and correlations**: two planes with the same pattern =
correlated features; a plane with a single smooth gradient = the feature is
mapped onto one grid direction. The primary ID column is masked during
training and gets no plane.

### Pie Maps (categorical dimensions only)

![Pie Map](examples/maps/pie-map_quality.png)
For each categorical column, a pie chart per neuron with the distribution of
its samples' categories. Coherent single-color regions = the map separates
the categories; mixed pies on a boundary are expected. The S-Curve example
set has no categorical column, example is from WineQuality dataset, `quality` dimension.

## Part 2 — Training plots

### MQE Evolution

![MQE Evolution](examples/plots/mqe_evolution.png)

Mean quantization error over training iterations with the best value marked.
Should fall steeply early and flatten; a rising tail or wild oscillation late
in training means the LR/radius schedule is too aggressive.

### Learning Rate & Radius Decay

![Learning Rate](examples/plots/learning_rate_decay.png)
![Radius](examples/plots/radius_decay.png)

The effective schedules actually used (including LSTM-controller
interventions when enabled — the controller modulates decay *speed*, the
curves must still always decrease). Domain constraint: rising LR/radius would
destroy the organization.

### Batch Size Growth

![Batch Size](examples/plots/batch_size_growth.png)

Samples processed per iteration — the hybrid training mechanism (sections ×
batch percent). Static here; a `*-growth` schedule shows the
stochastic→deterministic transition.

## Part 3 — Topology projections

Generated by `plot_som_topology.py` (see `RUN.md` for commands, the
`--isomap-fit` modes, and the full interpretation guide). The SOM grid
(neurons + edges) is projected into the data space next to the training
samples; **orange dashed edges** are the longest 15 % — either true
topological errors or projection artifacts, depending on the method.

### PCA — linear baseline

![PCA 2D](examples/topo/topology_2d_pca.png)
![PCA 3D](examples/topo/topology_3d_pca.png)

Fast linear view; the S-profile of the manifold is clearly visible. ⚠ A clean
PCA grid is **not** proof of manifold adherence — a SOM spanning a flat plane
through the Swiss Roll also looks clean here (`issues.md` #23). Use it for the
overall shape, not for the verdict.

### ISOMAP — geodesic verification view

![ISOMAP 2D](examples/topo/topology_2d_isomap.png)
![ISOMAP 3D](examples/topo/topology_3d_isomap.png)

Preserves geodesic (along-the-manifold) distances — the primary visual
verification. A manifold-following SOM appears as a coherent unfolded sheet
(this example); a layer-bridging SOM scatters with crossing edges. Two fit
modes exist (`--isomap-fit joint|data`) — the data fit is the honest view
when the joint embedding collapses (`issues.md` #23).

### UMAP / t-SNE — cluster views, not grid verification

![UMAP 2D](examples/topo/topology_2d_umap.png)
![t-SNE 2D](examples/topo/topology_2d_tsne.png)

⚠ Neighbor-embedding methods: they preserve only local neighborhoods and tear
continuous manifolds into fragments — the crossing edges in these examples
are **projection artifacts, not SOM errors** (`issues.md` #24). Useful only
for cluster-structured data (blobs); never judge grid coherence from them.

![UMAP 3D](examples/topo/topology_3d_umap.png)

### Compare grid — one-image summary

![Compare](examples/topo/topology_compare_pca_isomap.png)

2×2: (training data | SOM weights) × (PCA | ISOMAP), points colored by the
ground-truth manifold parameter. The fastest way to show "data structure vs
what the map learned" side by side — the standard figure for ablation
reports.

### Raw & interactive (no static example)

- `--projection raw` — **no transformation at all**; for benchmark datasets
  with 2–3 active dims this is the ground-truth view of the actual data space.
- `--html` / `--html3d` — interactive Plotly; the 3D variant is **rotatable by
  dragging**, which matters because a single fixed viewpoint can hide whether
  the grid is a clean sheet or a folded one. Files:
  `topology_interactive_<method>.html`, `topology_interactive_3d_<method>.html`.

Visual inspection shows *where* something breaks; the numbers come from
`app/tools/verify_topology.py` (local/global metric pair, PASS/WARN/FAIL —
see `RUN.md`).
