# Benchmark Verification — SOM Correctness Evidence

Status of the ground-truth benchmark battery (2026-06-11): what each dataset
verifies, the measured verdicts, and the resulting decision about SOM
correctness. This is the citable baseline for the ablation study (A3/A4 in
`docs/global/ABLATION_STUDY.md`). Methodology and commands: `RUN.md`
(benchmark generation, `verify_topology.py`, topology plots); tool
calibration history: `issues.md` #23–#30.

## Decision (summary)

**The SOM implementation is mathematically correct and preserves topology
wherever the manifold allows it.** Evidence:

- Local organization is correct on *every* benchmark (kNN grid→param R²
  0.75–1.00, per-neuron ANOVA ≥ 0.81 wherever ground truth exists).
- The easier unfoldable manifold (S-Curve) passes — a computational error in
  BMU search, weight update, or neighborhood would break exactly this case.
- The negative control (uniform cube) stays clean — the system does not
  invent structure.
- The two global FAILs (Swiss Roll, Helix) are a documented *property of
  Euclidean BMU matching on layered manifolds* in single deterministic runs,
  not a code defect — and standard metrics (MQE, TE, T&C) look healthy on
  both, which is itself a key thesis finding (deceptive metrics).

No drawing or computation error remains in the topology visualization chain;
the three issues found during this verification were all in the *evaluation
tools*, not in the SOM (see "Tool fixes" below).

## The benchmark datasets

All generated by `app/tools/generate_benchmark.py` (deterministic,
`random_seed: 42`) with a ground-truth file and a tuned `config-som.json`
per dataset. Each targets one property:

| Dataset | Structure | Map | Verifies |
|---|---|---|---|
| **SwissRoll** | 2D sheet rolled into a 3-layer spiral in 3D; param `t` (unrolling), `height` | 15×15 hex | Manifold unfolding — hard case: spiral layers are close in Euclidean space, far along the manifold |
| **SCurve** | 2D sheet bent into an S in 3D; param `t`, `height` | 15×15 hex | Manifold unfolding — easier case (no layering). A SOM failing here has a real defect |
| **Helix** | 1D curve coiled in 3D (3 turns); param `t` | 1×100 rect | Chain ordering along a curve — 1D analogue of the Swiss Roll (coils are vertically adjacent) |
| **Torus** | Closed 2D manifold in 3D; params `u`, `v` (angles) | 15×15 hex | Documented expected imperfection: a planar sheet cannot wrap a closed manifold |
| **Blobs** | 5 isotropic Gaussian clusters; integer `label` | 15×15 hex | Cluster separation (label ground truth → merged ARI, purity) |
| **NoisyPlane** | 2D signal (`u`, `v`) + 4 pure-noise numeric dims | 15×15 hex | Degradation caused by noise dimensions — *measurement input* for the preprocessing ablation (numeric noise is intentionally not excluded by preprocessing) |
| **UniformCube** | 3D uniform noise, no structure, no ground truth by design | 15×15 hex | **Negative control** — the system must not invent clusters (uniform hits, low Gini) |
| **SpaceFilling** | Uniform 2D points, no ground-truth parameter by design | 1×100 rect | Chain winding without self-crossings (computed directly on the weight polyline) |

## How to read the FAILs (thesis framing)

Swiss Roll and Helix fail for the same reason: **Euclidean BMU matching has no
notion of geodesic distance**, so when manifold layers are closer in ambient
space than along the manifold, the map bridges them. This is a property of
the classic SOM algorithm itself (single deterministic run, hand-tuned
config), not of this implementation — and it is invisible to MQE,
topographic error, and trustworthiness/continuity, all of which score
"healthy" on both failing runs. The benchmark battery turns this into
quantified, reproducible evidence:

> ideal mapping ρ ≈ 0.98 → bent-but-correct S-Curve 0.52 (WARN) →
> folded Swiss Roll 0.31 / Helix 0.16 (FAIL)

**Open question (ablation A3):** do EA-optimized configurations or multi-seed
statistics overcome the layering, or is it a principled limit to document?
The deterministic single runs above are the baseline for that comparison.

## Tool fixes made during this verification (issues #26–#30)

None of the problems found were in the SOM math; all were in the harness
around it:

- **#26** `map_type: "rect"` crashed `update_weights` (branch tested the enum
  by the opposite exclusion than the rest of the codebase).
- **#27** `compute_spatial_stats` crashed on 1×N chain maps (`np.gradient`
  needs ≥ 2 elements per axis).
- **#28** raw neuron-level ARI condemned a perfect blobs map (granularity
  mismatch); verdict now uses dominant-label-merged ARI + purity.
- **#29** compare-plot colours were row-index noise; now coloured by the
  ground-truth parameter (auto-discovered), making colour continuity along
  the grid the main visual folding diagnostic.
- **#30** space-filling verification ended at NO-GROUNDTRUTH; added the chain
  self-crossing count.

The full test suite covering the SOM core, preprocessing, analysis,
visualization, persistence, and these tools passes (270 tests, see
`TESTING.md`).
